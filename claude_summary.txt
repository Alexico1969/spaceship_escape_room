OVERVIEW
--------
A Flask-based text-adventure escape room game with 20 levels.
Players type commands to explore rooms, solve puzzles and escape Lyra Station.
Built for use in a classroom setting (Molloy HS). Teacher: avanwinkel@molloyhs.org.
Players who complete all 20 levels receive a unique code to email the teacher.


TECH STACK
----------
- Python / Flask
- Flask-Session (filesystem sessions)
- SQLite (database.db) — single 'users' table
- Werkzeug password hashing
- python-dotenv for SECRET_KEY
- Jinja2 templates
- Vanilla CSS + JS (no frameworks)
- Managed via Pipenv (Pipfile) + requirements.txt

DATABASE
--------
Single table: users
  name TEXT, email TEXT, username TEXT, password TEXT (hashed),
  level INTEGER, score INTEGER, inventory TEXT (comma-separated)

Admin user: username='admin' — grants access to /admin dashboard.


SESSION VARIABLES
-----------------
user, level, score, inventory (list), objects (dict), door_status,
room_type, sequence_step, wrong_attempts, log_slid, key_on_log,
key_lost, end_code, new (bool — controls intro video display)


ROUTES
------
GET/POST /           — main game loop
GET/POST /next_level — level transition (hidden field prevents back-button skip)
GET      /win        — win screen (legacy, not used)
GET      /admin      — admin dashboard (admin only)
POST     /admin/delete/<username>
POST     /admin/reset/<username>
GET/POST /login, /register, /logout
GET      /dump1      — legacy raw dump (admin only)


ROOM TYPES (in database.py Room class)
---------------------------------------
code        — enter a numeric or text code
riddle      — answer a free-text riddle
inventory   — use a specific item from inventory
combo       — combine two items → new item → use result
sequence    — type commands in correct order (e.g. switch a, switch c)
retrieval   — multi-step physical puzzle (level 19 key-under-door)
pod         — (legacy, replaced by retrieval in level 19)
end_sequence — final room: plays video, shows completion code
end         — (legacy)


ROOM CLASS FIELDS
-----------------
level, name, description, type, furniture (comma str), objects (dict),
expected_output, required_item, required_item_alt (list of alternatives),
combo_items (list), combo_result, sequence (list of commands),
must_take (list — items required before exit), combo_key (e.g. "13"),
combo_wrong_key_msg, take_requirements (dict item→required),
use_effect ("unlock" or "reveal"), use_reveal_updates (dict),
use_reveal_message, max_attempts (0=no lockout), door_look_msg


OBJECTS DICT CONVENTION
------------------------
"key": "value"          → takeable item
"key": "<description>"  → untakeable, observable only
"key": ""               → slot is empty (item was taken)


PROCESS.PY COMMAND FLOW (in order)
------------------------------------
1. exit / leave / open door  — checks door_status, must_take, required_item
2. help
3. read <x>              → redirected to "look at <x>"
4. look / look around
5. look at / examine / look <thing>  — searches objects keys then values then inventory
6. take / get / pick up  — checks take_requirements before allowing
7. RETRIEVAL handler      — level 19 multi-step puzzle (log_slid, key_on_log, key_lost)
8. open <thing>           — door goes to exit handler; other objects open normally
9. combine / natural language combo detection
10. use / insert / place / put / slot / apply / install / fix / repair / attach / connect / fit
11. sequence rooms
12. pod rooms (legacy)
13. end_sequence room
14. code / riddle / inventory type answer check (with optional max_attempts lockout)
15. flavour responses (hit, break, run, etc.)


OVERRIDE COMMANDS (tester only)
---------------------------------
**back          — go to previous level (resets room state)
**remove <item> — remove item from inventory
(Note: **forward was intentionally REMOVED — players cannot skip ahead)


INTRO / FINALE VIDEO
---------------------
Intro: static/Spaceship_Escape_Room_Intro_v1.mp4
  - Shows on first login (session['new'] = True)
  - Has X button to close (pauses + hides)
  - Submit button hidden until video ends or X is clicked
  - Set to 70% of screen width via .vid CSS class

Finale: static/rooms/finale.mp4
  - Plays on level 20
  - Code revealed after video ends
  - Code format: USERNAME-DDMM-HHMM-XX (XX = random 2 digits)
  - Stored in session['end_code'], generated once per login


END CODE
--------
Generated in flask_app.py home route when level == 20.
Format: {username[:6].upper()}-{DDMM}-{HHMM}-{random 2 digits}
Stored in session so it doesn't change on page refresh.
Reset on login (session.pop('end_code', None)).

KNOWN QUIRKS / DECISIONS
--------------------------
- Passwords are hashed (werkzeug). Existing plaintext passwords won't work after migration.
- session['objects'] is set on login/level-up from rooms[level].objects.
  If room data changes while a player is mid-level, they need **back + re-enter to reload.
- Article stripping (' the ', ' a ', ' an ') happens BEFORE most handlers.
  original_inp is preserved before stripping (needed for switch 'a' in level 6).
- "cypher" is normalised to "cipher" at top of process().
- use_verbs tuple: use, insert, place, put, slot, apply, install, fix, repair, attach, connect, fit
- The 'decoded log' (room 9) stays in inventory after use — needed for level 19 retrieval puzzle.
- Level 17 requires fitted pipe, wrench, OR multi-tool (required_item_alt list).
- Level 16 has max_attempts=5 lockout — only room with this feature.
- Back-button double-submit on /next_level prevented via hidden 'current_level' field.
- Video buttons all have type="button" to prevent accidental form submission.
- *.mp4 and .env and database.db are in .gitignore (not in GitHub repo).
- Admin route is /admin (not /dump1 which is legacy).
- The win.html page exists but is not used — level 20 is end_sequence, not a win redirect.
